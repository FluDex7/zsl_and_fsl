# Часть I. Фундамент

## 1. Что такое Zero-Shot и Few-Shot Learning

### 1.1 Проблема, которую мы решаем

Стандартный классификатор на нейронных сетях работает так: мы собираем тысячи размеченных изображений (или текстов) для каждого класса, обучаем модель, и она умеет различать **ровно те классы**, на которых обучалась. Появился новый класс — нужно заново собрать данные, разметить, переобучить.

Это создаёт три практические проблемы:

**Дефицит данных.** Разметка стоит дорого. Для медицинских снимков нужны эксперты-врачи. Для редких видов животных или растений данных может просто не существовать — в мире около 400 000 видов растений и 30 000 видов птиц, и собрать по тысяче фотографий каждого вида невозможно.

**Динамические классы.** В интернет-магазине каждый день появляются сотни новых товаров. В системе видеонаблюдения нужно искать конкретного человека по описанию. Переобучать модель каждый раз — нереалистично.

**Скорость адаптации.** Иногда нужно перейти от идеи к рабочему классификатору за минуты, а не за недели обучения.

Zero-Shot Learning (обучение без примеров) и Few-Shot Learning (обучение по нескольким примерам) — два подхода, которые решают эти проблемы.

### 1.2 Определения

**Zero-Shot Learning (ZSL)** — способность модели распознавать классы, которых она **никогда не видела** при обучении. Не просто новые изображения знакомых классов, а **целые новые классы**. Вместо обучающих примеров модель получает **описание** нового класса — через атрибуты, текст, или эмбеддинги.

**Few-Shot Learning (FSL)** — способность модели научиться распознавать новый класс, увидев всего **1–16 примеров**. Человек легко узнаёт утконоса, увидев одну фотографию. Стандартной нейросети для этого нужны тысячи.

Между ними — непрерывный спектр:

```
Zero-Shot (0 примеров) → One-Shot (1) → Few-Shot (2–16) → Supervised (тысячи+)
```

### 1.3 Формальная постановка задачи ZSL

Чтобы точно понимать, что мы решаем, запишем задачу математически.

У нас есть обучающая выборка: набор пар (изображение, класс):

$$S = \{(x_n, y_n),\ n = 1 \ldots N\}, \quad y_n \in Y_{seen}$$

где $Y_{seen}$ — множество **видимых** (seen) классов, на которых модель обучается.

Каждый класс $y$ описан **семантическим вектором** $\varphi(y)$ — это может быть вектор атрибутов (85 бинарных признаков вроде «полосатый», «четвероногий»), word2vec-эмбеддинг названия класса, или текстовое описание.

Есть множество **невидимых** (unseen) тестовых классов $Y_{unseen}$, причём видимые и невидимые классы **не пересекаются**:

$$Y_{seen} \cap Y_{unseen} = \varnothing$$

**Задача:** построить классификатор $f: X \to Y_{unseen}$, который относит новое изображение к одному из невидимых классов, **не имея ни одного обучающего примера** этих классов.

Как это возможно? Через **функцию совместимости** (compatibility function) — она оценивает, насколько изображение $x$ «подходит» к описанию класса $y$:

$$f(x) = \arg\max_{y \in Y_{unseen}} F(x, y; W)$$

Модель выбирает тот unseen-класс, чьё описание лучше всего «совместимо» с изображением. В простейшем (и самом распространённом) случае функция совместимости — **билинейная**:

$$F(x, y; W) = \theta(x)^\top \cdot W \cdot \varphi(y)$$

Здесь:
- $\theta(x) \in \mathbb{R}^{d_v}$ — визуальное представление изображения (например, 2048-мерный вектор из ResNet-101)
- $\varphi(y) \in \mathbb{R}^{d_s}$ — семантическое описание класса (вектор атрибутов или word2vec)
- $W \in \mathbb{R}^{d_v \times d_s}$ — обучаемая матрица проекции, которая «связывает» визуальное и семантическое пространства

Смысл: матрица $W$ учится при обучении на seen-классах, а при тестировании применяется к unseen-классам. Это работает, потому что семантическое пространство **общее** — и для seen, и для unseen классов (например, атрибут «полосатый» одинаково определён для всех животных).

### 1.4 Обобщённый Zero-Shot Learning (Generalized ZSL, GZSL)

В обычном ZSL при тестировании модель выбирает только из unseen-классов. Но в реальной жизни нам встречаются и старые, и новые объекты. GZSL — более реалистичная постановка, где при тестировании пространство поиска включает **и видимые, и невидимые классы**:

$$Y_{test} = Y_{seen} \cup Y_{unseen}$$

Это существенно сложнее, потому что модель **систематически предпочитает** видимые классы — она на них обучалась и «знает» их лучше. Для оценки GZSL используется **гармоническое среднее** точности на видимых ($Acc_S$) и невидимых ($Acc_U$) классах:

$$H = \frac{2 \cdot Acc_S \cdot Acc_U}{Acc_S + Acc_U}$$

Почему именно гармоническое среднее, а не обычное? Потому что оно **наказывает за дисбаланс**: если модель идеально классифицирует seen-классы (100%), но полностью проваливается на unseen (0%), обычное среднее было бы 50%, а гармоническое — 0%. Это заставляет модель балансировать между двумя задачами.

### 1.5 Формальная постановка задачи FSL

Few-Shot Learning формулируется как **N-way K-shot задача**:

- Дано **N классов** (ways) — это новые, не виденные при обучении классы
- Для каждого класса дано **K размеченных примеров** (shots), обычно K = 1 или 5
- Эти N×K примеров образуют **support set** (опорное множество) $S$
- Есть **query set** (запросное множество) $Q$ — новые примеры, которые нужно классифицировать

Задача: для каждого примера из query set определить, к какому из N классов он относится, используя только K примеров каждого класса.

**Эпизодическое обучение** (episodic training) — ключевой принцип FSL: модель обучается не на обычных мини-батчах, а на **эпизодах**, каждый из которых имитирует N-way K-shot задачу. На каждом шаге обучения случайно выбираются N классов из обучающего множества, для каждого берётся K примеров в support и несколько в query, и модель учится решать эту «мини-задачу». Идея: если при обучении условия совпадают с условиями тестирования, модель лучше обобщается.

Связь ZSL и FSL очевидна: при K = 0 FSL превращается в ZSL.

## 2. Как к этому пришли: история идеи

### 2.1 Эпоха атрибутов (2006–2013): первые шаги

#### Откуда взялась идея one-shot learning

В 2006 году Ли Фей-Фей (одна из создателей ImageNet) поставила вопрос: **можно ли научить компьютер узнавать новый объект по одному примеру?** Люди делают это легко — увидев одну фотографию утконоса, мы узнаем его в зоопарке. Почему компьютер не может?

Идея решения — **байесовский перенос знаний**: если модель уже изучила много категорий (кошки, собаки, лошади), она может использовать это знание как **априорное распределение** при обучении новой категории. Формально: вместо обучения с нуля, модель обновляет уже имеющееся представление о «типичном объекте» по одному наблюдению.

Это была первая работа, показавшая, что **знание о предыдущих категориях переносимо** на новые. До этого каждую категорию учили с нуля.

> Источник: Fei-Fei, Fergus, Perona — *One-Shot Learning of Object Categories* (TPAMI, 2006)

#### Рождение Zero-Shot Learning: атрибуты как мост

В 2009 году Кристоф Ламперт с коллегами сделал следующий логический шаг: а что если примеров нового класса нет **вообще**? Можно ли распознать объект, ни разу его не видев?

Ключевая идея оказалась простой и элегантной: **если мы не можем дать модели примеры нового класса, мы можем дать ей описание этого класса** через свойства (атрибуты), которые модель уже умеет распознавать.

Представьте, что вы биолог и видите незнакомое животное. У вас нет фотокаталога, но есть таблица:

| Вид | Полосатый | Четвероногий | Водоплавающий | Чёрно-белый |
|-----|-----------|-------------|---------------|-------------|
| Зебра | ✓ | ✓ | ✗ | ✓ |
| Пингвин | ✗ | ✗ | ✓ | ✓ |
| Тигр | ✓ | ✓ | ✗ | ✗ |

Вы смотрите на животное: полосатое? да. Четвероногое? да. Водоплавающее? нет. Чёрно-белое? да. Сравниваете с таблицей → зебра!

Модель делает ровно то же самое: обучается распознавать 85 отдельных атрибутов (цвет, форма, среда обитания, тип покрова и т.д.) на видимых классах, а потом применяет эти классификаторы к новому изображению и сравнивает результат с атрибутными описаниями невидимых классов.

Ламперт и коллеги создали датасет **Animals with Attributes (AwA)** — 50 классов животных, описанных через 85 атрибутов — и показали, что подход реально работает.

> Источник: Lampert, Nickisch, Harmeling — *Learning to Detect Unseen Object Classes by Between-Class Attribute Transfer* (CVPR 2009, расширенная версия TPAMI 2014)

#### Семантические выходные коды

Параллельно с Лампертом, группа Палатуччи, Померло, Хинтона и Митчелла предложила более общий взгляд: описывать классы не через ручные атрибуты, а через **«семантические коды»** — координаты в любом семантическом пространстве. Они дали **первые теоретические гарантии** для ZSL: формально доказали, что при достаточно хорошем семантическом пространстве ZSL может работать с гарантированной точностью (PAC-bound).

> Источник: Palatucci, Pomerleau, Hinton, Mitchell — *Zero-shot Learning with Semantic Output Codes* (NeurIPS 2009)

### 2.2 Эпоха эмбеддингов (2013–2016): от ручных атрибутов к автоматическим

#### Проблема атрибутов

Атрибутный подход Ламперта работал, но имел серьёзное ограничение: **атрибуты нужно придумывать и размечать вручную**. Для 50 животных придумать 85 атрибутов — ещё терпимо. Для 1000 классов ImageNet — уже мучительно. Для 20 000+ классов — невозможно.

Вопрос: можно ли заменить ручные атрибуты чем-то автоматическим?

#### DeViSE: Word2Vec вместо атрибутов

В 2013 году команда Google предложила элегантное решение: использовать **word2vec-эмбеддинги** названий классов вместо ручных атрибутов. Word2vec — модель, которая обучается на огромных текстовых корпусах и выдаёт для каждого слова числовой вектор, причём семантически похожие слова получают близкие векторы («кошка» и «собака» ближе, чем «кошка» и «самолёт»).

Архитектура DeViSE:
- **Визуальный энкодер:** CNN (AlexNet) извлекает из изображения 4096-мерный вектор признаков
- **Семантический энкодер:** Word2Vec (Skip-gram) даёт для названия класса 500/1000-мерный вектор
- **Обучаемая проекция:** линейное преобразование $M$ из визуального пространства в семантическое
- **Функция потерь:** hinge rank loss — правильный класс должен быть ближе к изображению, чем неправильные

[СХЕМА: Архитектура DeViSE — Изображение → CNN → 4096-dim → Linear M → 500-dim ↔ Word2Vec «cat» 500-dim]

> Источник: Frome et al. — *DeViSE: A Deep Visual-Semantic Embedding Model* (NeurIPS 2013)

#### ALE и SJE: улучшения проекции

После DeViSE появилось несколько методов, которые улучшали качество проекции из визуального в семантическое пространство. Все они используют одну и ту же **билинейную функцию совместимости**:

$$F(x, y; W) = \theta(x)^\top \cdot W \cdot \varphi(y)$$

Разница — только в **функции потерь** при обучении матрицы $W$:

| Метод | Функция потерь | Идея |
|-------|---------------|------|
| DeViSE | Hinge rank loss | Правильный класс ближе неправильного на отступ Δ |
| ALE (Attribute Label Embedding) | WARP ranking loss | Больший штраф за ошибки вверху ранжированного списка |
| SJE (Structured Joint Embedding) | Structured SVM loss | Штраф пропорционален «серьёзности» ошибки |

Все три метода решают одну задачу: научить матрицу $W$ так проецировать визуальные признаки, чтобы изображение зебры оказалось ближе к слову «зебра», чем к слову «лошадь» в семантическом пространстве.

> Источники: Akata et al. — *Label-Embedding for Image Classification* (TPAMI 2016); Akata et al. — *Evaluation of Output Embeddings for Fine-Grained Image Classification* (CVPR 2015)

### 2.3 Эпоха генеративных моделей (2017–2020): если нет данных — сгенерируем

#### Идея

Предыдущие подходы (DAP, DeViSE, ALE) учились **проецировать** изображения в семантическое пространство. Но проекция, обученная на seen-классах, плохо работает для unseen-классов — это называется **domain shift** (сдвиг домена).

Группа Юнцинь Сиань предложила радикально другой подход: если проблема ZSL в том, что для unseen-классов нет обучающих данных, давайте их **сгенерируем**. Причём генерировать не сами изображения (это слишком сложно), а их **CNN-признаки** — 2048-мерные векторы из ResNet-101.

#### f-CLSWGAN: как это работает

f-CLSWGAN — это условный генератор на базе Wasserstein GAN. Он принимает на вход случайный шум $z$ и описание класса $c(y)$ (вектор атрибутов), и выдаёт синтетический вектор CNN-признаков.

Архитектура генератора и дискриминатора — простые MLP (многослойные перцептроны) с одним скрытым слоем (4096 нейронов).

[СХЕМА: f-CLSWGAN — Шум z + Описание класса c(y) → Генератор G (MLP) → Синтетический feature x̃ (2048-dim) ↔ Дискриминатор D (MLP) ← Реальный feature x]

Модель обучается минимизировать комбинацию двух функций потерь:

$$\min_G \max_D \; L_{WGAN} + \beta \cdot L_{CLS}$$

**Первый компонент — $L_{WGAN}$** (Wasserstein loss с gradient penalty): стандартная GAN-потеря, которая заставляет сгенерированные признаки быть неотличимыми от реальных:

$$L_{WGAN} = \mathbb{E}[D(x, c(y))] - \mathbb{E}[D(\tilde{x}, c(y))] - \lambda \mathbb{E}\left[(\|\nabla_{\hat{x}} D(\hat{x}, c(y))\|_2 - 1)^2\right]$$

Здесь:
- $x$ — реальный CNN-признак (2048-dim вектор из ResNet-101)
- $\tilde{x} = G(z, c(y))$ — сгенерированный CNN-признак
- $c(y)$ — семантическое описание класса (вектор атрибутов)
- $D(\cdot)$ — дискриминатор, оценивающий «реальность» признака
- $\hat{x}$ — интерполяция между реальным и сгенерированным (для gradient penalty)
- $\lambda = 10$ — вес штрафа за градиент

**Второй компонент — $L_{CLS}$** (классификационный loss): гарантирует, что сгенерированные признаки не просто «реалистичны», но и **содержат информацию о классе**:

$$L_{CLS} = -\mathbb{E}_{\tilde{x} \sim p_G}\left[\log P(y|\tilde{x}; \theta)\right]$$

Здесь $P(y|\tilde{x}; \theta)$ — предобученный softmax-классификатор. Он проверяет: можно ли по сгенерированному признаку $\tilde{x}$ правильно определить класс $y$?

$\beta = 0.01$ — маленький вес, чтобы классификационный loss не подавлял основной GAN-loss.

**Как это используется для ZSL:**

1. Обучаем f-CLSWGAN на данных seen-классов (реальные CNN-признаки + атрибуты)
2. Для каждого unseen-класса $u$ берём его атрибутный вектор $c(u)$ и генерируем N синтетических CNN-признаков: $\tilde{x}_i = G(z_i, c(u))$
3. Обучаем обычный softmax-классификатор на **объединении** реальных (seen) и сгенерированных (unseen) данных
4. Применяем этот классификатор к новым изображениям

**Ключевой эффект: f-CLSWGAN конвертирует ZSL в обычное обучение с учителем** (supervised learning). Вместо того чтобы решать сложную задачу «распознай без примеров», мы решаем простую «распознай с примерами» — просто примеры синтетические.

> Источник: Xian et al. — *Feature Generating Networks for Zero-Shot Learning* (CVPR 2018)

#### f-VAEGAN-D2: объединение VAE и GAN

Следующий шаг — объединить VAE (вариационный автоэнкодер) и GAN. В f-VAEGAN-D2 генератор GAN и декодер VAE **разделяют параметры**, что улучшает стабильность обучения. Добавляется второй «безусловный» дискриминатор, который обучается на неразмеченных данных (потенциально включающих unseen-классы), обеспечивая сигнал о «маргинальном» распределении признаков.

> Источник: Xian et al. — *f-VAEGAN-D2: A Feature Generating Framework for Any-Shot Learning* (CVPR 2019)

### 2.4 Эпоха фундаментальных моделей (2021–настоящее): CLIP и революция

#### Что изменилось

До 2021 года каждый ZSL-метод был «штучным»: DAP требовал ручных атрибутов, DeViSE — word2vec, f-CLSWGAN — обучения GAN. Все они работали на небольших датасетах (AwA: 50 классов, CUB: 200 видов птиц).

CLIP изменил правила: **обучи одну модель на 400 миллионах пар (изображение, текст) из интернета — и она из коробки делает zero-shot классификацию на любых классах, описанных текстом.** Не нужны ни атрибуты, ни специальные алгоритмы, ни GAN. Просто напиши «A photo of a zebra» — и модель найдёт зебру.

Подробный разбор архитектуры CLIP — в разделе 3.4.

## 3. Zero-Shot Learning в Computer Vision

### 3.1 Метод прямого предсказания атрибутов (Direct Attribute Prediction, DAP)

#### Как это работает

DAP — первый и наиболее интуитивно понятный метод ZSL. Идея: каждый класс описывается набором **семантических атрибутов** — бинарных свойств вроде «полосатый», «четвероногий», «водоплавающий».

Процесс состоит из двух этапов:

**Этап обучения:** для каждого из M атрибутов обучается отдельный бинарный классификатор. Например, классификатор «полосатый» обучается на всех изображениях seen-классов: для зебры и тигра выход = 1, для медведя и слона = 0. После обучения есть M классификаторов, каждый из которых умеет определять один атрибут по изображению.

**Этап тестирования:** новое изображение прогоняется через все M классификаторов. Получаем вектор предсказанных атрибутов. Сравниваем его с описаниями всех unseen-классов и выбираем наиболее похожий.

[СХЕМА: DAP Pipeline — Изображение x → [M атрибутных классификаторов: p(a₁|x), p(a₂|x), ..., p(aₘ|x)] → Предсказанный вектор атрибутов → Сравнение с aʸ каждого unseen-класса → argmax → Класс ŷ]

#### Математика DAP

Каждый unseen-класс $y$ описан бинарным вектором атрибутов:

$$a^y = (a_1^y, a_2^y, \ldots, a_M^y) \in \{0, 1\}^M$$

Например, для зебры: $a^{\text{зебра}} = (1, 1, 0, 1, \ldots)$ — полосатая (1), четвероногая (1), не водоплавающая (0), чёрно-белая (1), ...

Для изображения $x$ каждый классификатор выдаёт вероятность соответствующего атрибута: $p(a_m | x)$.

Предсказание класса — это поиск unseen-класса, чей вектор атрибутов **наиболее вероятен** при данном изображении:

$$\hat{y} = \arg\max_{y \in Y_{unseen}} \prod_{m=1}^{M} p(a_m | x)^{a_m^y} \cdot (1 - p(a_m | x))^{(1 - a_m^y)}$$

Что означает эта формула? Для каждого unseen-класса $y$ мы перемножаем вероятности:
- Если атрибут $m$ **есть** у класса ($a_m^y = 1$): берём $p(a_m | x)$ — вероятность, что модель видит этот атрибут на изображении
- Если атрибута **нет** ($a_m^y = 0$): берём $1 - p(a_m | x)$ — вероятность, что модель **не** видит этот атрибут

Произведение этих вероятностей — насколько хорошо изображение «совпадает» с описанием класса.

**В логарифмической форме** (для вычислительной стабильности — произведение многих чисел < 1 быстро обнуляется):

$$\hat{y} = \arg\max_{y} \sum_{m=1}^{M} \left[ a_m^y \cdot \log p(a_m|x) + (1 - a_m^y) \cdot \log(1 - p(a_m|x)) \right]$$

**Важное допущение:** формула предполагает **условную независимость** атрибутов при данном изображении. Это упрощение — в реальности «полосатый» и «четвероногий» коррелируют (полосатые животные чаще четвероногие). Но допущение делает вычисления простыми и на практике работает удовлетворительно.

#### Косвенное предсказание атрибутов (Indirect Attribute Prediction, IAP)

IAP — альтернативный подход, где атрибуты предсказываются **не напрямую** из изображения, а через промежуточный шаг.

Сначала обучается обычный мультиклассовый классификатор на seen-классах: $p(y_s | x)$ — вероятность каждого seen-класса. Затем атрибуты вычисляются через **маргинализацию** по seen-классам:

$$p(a_m | x) = \sum_{y_s \in Y_{seen}} p(a_m | y_s) \cdot p(y_s | x)$$

Здесь:
- $p(y_s | x)$ — вероятность, что изображение принадлежит seen-классу $y_s$ (из мультиклассового классификатора)
- $p(a_m | y_s)$ — вероятность атрибута $m$ для seen-класса $y_s$ (известна из таблицы атрибутов)

Логика: если модель считает, что на изображении тигр (seen-класс), а у тигра атрибут «полосатый» = 1, то изображение, вероятно, полосатое.

После получения $p(a_m | x)$ для всех атрибутов, применяется та же формула выбора класса, что и в DAP.

**Разница DAP vs IAP:** DAP предсказывает атрибуты напрямую из изображения (один шаг). IAP — сначала предсказывает seen-класс, потом выводит атрибуты через таблицу (два шага). DAP обычно работает лучше, потому что не зависит от точности мультиклассовой классификации.

#### Ограничения атрибутного подхода

1. **Дорогая аннотация.** 85 атрибутов × 50 классов — это 4250 ручных аннотаций. Для тысяч классов нереалистично.
2. **Субъективность.** Разные аннотаторы по-разному определяют «большой» или «пушистый».
3. **Грубость описания.** 85 бинарных атрибутов — очень грубое приближение. Два класса могут иметь одинаковые атрибуты, но выглядеть совершенно по-разному.
4. **Допущение независимости** атрибутов не выполняется в реальности.


### 3.2 Embedding-подходы (Подходы на основе эмбеддингов, совместных вложений)

Все embedding-подходы решают одну задачу: **научить проекцию из визуального пространства в семантическое**, чтобы изображение зебры оказалось рядом с описанием «зебры».

Общая формула совместимости (одинаковая для DeViSE, ALE, SJE):

$$F(x, y; W) = \theta(x)^\top \cdot W \cdot \varphi(y)$$

где $\theta(x)$ — CNN-признаки изображения (ResNet-101, 2048 измерений), $\varphi(y)$ — семантическое описание класса (атрибуты, word2vec или GloVe), $W$ — обучаемая матрица проекции.

Обучение: $W$ оптимизируется так, чтобы $F(x, y_{correct}; W) > F(x, y_{wrong}; W)$ — совместимость изображения с правильным классом выше, чем с неправильным. Разница между методами — как именно формулируется штраф за нарушение этого условия (функция потерь).

При инференсе: для нового изображения вычисляется $F(x, y; W)$ для всех unseen-классов, выбирается класс с максимальным значением.


### 3.3 Генеративные подходы: f-CLSWGAN и f-VAEGAN-D2

Подробно разобраны в разделе 2.3 (история). Ключевая идея: **генерация синтетических CNN-признаков** для unseen-классов через условный GAN, что позволяет конвертировать ZSL в обычное обучение с учителем.

Результаты на стандартных бенчмарках показывают значительное улучшение, особенно в GZSL: f-CLSWGAN повышает гармоническое среднее $H$ на CUB с 34.4% до 49.7%.


### 3.4 CLIP: революция в zero-shot CV

#### Что такое CLIP и почему он изменил всё

CLIP (Contrastive Language-Image Pre-training, контрастивное предобучение на парах язык-изображение) — модель, обученная на ~400 миллионах пар (изображение, текст) из интернета. CLIP заменил весь предыдущий арсенал ZSL одним простым принципом: **естественный язык — универсальный интерфейс для описания классов.**

Вместо ручных атрибутов (DAP), word2vec-эмбеддингов (DeViSE) или генерации признаков (f-CLSWGAN), CLIP позволяет просто написать «A photo of a zebra» — и модель найдёт зебру на изображении. Без какой-либо специальной настройки.

Результат: **76.2% top-1 accuracy на ImageNet zero-shot** (модель ViT-L/14@336px) — это уровень supervised ResNet-50, обученного на 1.28 миллиона размеченных изображениях. Модель, ни разу не видевшая ImageNet при обучении, показывает такой же результат, как модель, обученная на нём.

#### Архитектура CLIP

CLIP состоит из **двух независимых энкодеров**: визуального (кодирует изображение в вектор) и текстового (кодирует текст в вектор). Они не связаны архитектурно (нет cross-attention), но обучаются совместно.

[СХЕМА: Полная архитектура CLIP — см. CLIP_Architecture_v2.tex]

**Визуальный энкодер (Image Encoder)** — Vision Transformer (ViT):

Для модели ViT-B/32:
1. Изображение 224×224×3 разрезается на патчи 32×32 пикселей → получается 7×7 = 49 патчей
2. Каждый патч проецируется через свёрточный слой Conv2d(3, 768, kernel=32, stride=32) → 49 векторов по 768 измерений
3. Добавляется обучаемый [CLS]-токен (один дополнительный вектор) → 50 токенов × 768
4. Прибавляются обучаемые позиционные эмбеддинги (positional embeddings) → 50 × 768
5. Проход через LayerNorm, затем 12 слоёв Transformer (каждый: LayerNorm → Multi-Head Self-Attention с 12 головами по 64 измерения → Residual → LayerNorm → MLP 768→3072→768 с QuickGELU → Residual)
6. Финальный LayerNorm, берётся вектор [CLS]-токена → 768-мерный вектор
7. Линейная проекция 768→512 + L2-нормализация → выход $I_e \in \mathbb{R}^{512}$

**Текстовый энкодер (Text Encoder)** — Transformer в стиле GPT:
1. Текст «A photo of a cat» токенизируется через BPE (словарь 49 408 токенов) → макс. 77 токенов
2. Token Embedding + Positional Embedding → 77 × 512
3. 12 слоёв Transformer с **каузальной маской** (masked self-attention): токен $i$ видит только токены $1 \ldots i$, 8 голов по 64 измерения, MLP 512→2048→512 с QuickGELU
4. Финальный LayerNorm, берётся hidden state на позиции [EOS]-токена → 512-мерный вектор
5. Линейная проекция 512→512 + L2-нормализация → выход $T_e \in \mathbb{R}^{512}$

#### Обучение CLIP: контрастивный loss

CLIP обучается на батче из $N$ пар (изображение, текст). Строится матрица косинусных сходств $N \times N$: каждое изображение сравнивается с каждым текстом.

$$\text{logits}_{ij} = \cos(I_e^{(i)}, T_e^{(j)}) \cdot \exp(\tau)$$

Здесь:
- $I_e^{(i)}$ — эмбеддинг $i$-го изображения (L2-нормализованный)
- $T_e^{(j)}$ — эмбеддинг $j$-го текста (L2-нормализованный)
- $\tau$ — обучаемый скаляр (температура), инициализируется как $\log(1/0.07) \approx 2.66$

Правильные пары — на диагонали ($i = j$). Loss — **симметричная кросс-энтропия**:

$$\mathcal{L} = \frac{1}{2}\left[ \underbrace{\text{CE}(\text{logits}, \text{labels}, \text{по строкам})}_{\text{для каждого изображения: найди правильный текст}} + \underbrace{\text{CE}(\text{logits}, \text{labels}, \text{по столбцам})}_{\text{для каждого текста: найди правильное изображение}} \right]$$

где $\text{labels} = [0, 1, 2, \ldots, N\!-\!1]$.

Почему loss **симметричный**? Первая часть учит: «для данного изображения найди правильный текст среди $N$». Вторая: «для данного текста найди правильное изображение среди $N$». Обе задачи нужны, чтобы пространство было **выровнено в обе стороны**.

Почему **температура обучаемая**? Она контролирует «остроту» softmax: маленькая $\tau$ → все пары равновероятны (модель неуверенна); большая $\tau$ → один чёткий пик (модель очень уверенна). Модель сама находит нужный уровень уверенности.

#### Zero-Shot инференс CLIP

При инференсе модель **не переобучается**. Алгоритм:

1. Для каждого класса создаём текстовый промпт: «A photo of a {class name}»
2. Кодируем все промпты текстовым энкодером → $K$ текстовых эмбеддингов $T_e^1, T_e^2, \ldots, T_e^K$
3. Кодируем тестовое изображение визуальным энкодером → $I_e$
4. Вычисляем косинусное сходство со всеми текстовыми эмбеддингами
5. Выбираем класс с максимальным сходством:

$$\hat{y} = \arg\max_k \cos(I_e, T_e^k)$$

**Prompt engineering** (инженерия промптов) существенно влияет на качество. Вместо одного шаблона можно использовать ансамбль: «A photo of a {class}», «A blurry photo of a {class}», «A photo of the large {class}» и т.д. (до 80 шаблонов в оригинальной статье). Эмбеддинги усредняются — это повышает accuracy.

> Источник: Radford, Kim et al. — *Learning Transferable Visual Models From Natural Language Supervision* (ICML 2021)


### 3.5 SigLIP: улучшение CLIP через sigmoid loss

SigLIP (Sigmoid Loss for Language-Image Pre-training) — модель с архитектурой, **идентичной CLIP**, но с другой функцией потерь.

**Проблема CLIP:** softmax-based контрастивный loss требует **глобальной нормализации** по всему батчу — каждый пример зависит от всех остальных в батче. Это: (а) ограничивает масштабирование; (б) требует больших батчей для хорошей работы.

**Решение SigLIP:** заменить softmax на **sigmoid** — каждая пара (image, text) оценивается **независимо**:

$$\mathcal{L}_{sigmoid} = -\frac{1}{|B|}\sum_{i}\sum_{j} \log \frac{1}{1 + e^{z_{ij} \cdot (-\tau \cdot \cos(I_e^{(i)}, T_e^{(j)}) + b)}}$$

Здесь:
- $z_{ij} = +1$ если пара $(i, j)$ совпадает (правильная пара), $z_{ij} = -1$ если не совпадает
- $\tau$ — обучаемая температура
- $b$ — обучаемый bias (смещение)

По сути это **бинарная кросс-энтропия** для каждой пары: «совпадают ли это изображение и этот текст?» — да/нет. Никакой глобальной нормализации.

**Преимущества SigLIP:**
- Работает лучше CLIP при **маленьких батчах** (важно при ограниченных ресурсах)
- Масштабируется до **очень больших батчей** (32K+)
- Превосходит CLIP по zero-shot accuracy на ImageNet при сравнимых ресурсах

> Источник: Zhai et al. — *Sigmoid Loss for Language Image Pre-Training* (arXiv:2303.15343, 2023)


### 3.6 OWL-ViT: Zero-Shot Object Detection (обнаружение объектов без обучения)

#### От классификации к локализации

CLIP классифицирует **всё изображение целиком**: «на этом изображении кошка». Но часто нужно не только определить **что** на изображении, но и **где** — найти и обвести объект рамкой (bounding box). Это задача **object detection** (обнаружение объектов).

OWL-ViT (Vision Transformer for Open-World Localization) — модель, которая адаптирует CLIP для zero-shot object detection **минимальными изменениями**.

#### Ключевая идея: каждый патч — потенциальный объект

В CLIP визуальный энкодер (ViT) превращает изображение в набор patch-токенов (один на каждый патч), но потом всё сводится к одному [CLS]-токену — одному вектору на всё изображение.

OWL-ViT **убирает этот финальный pooling** и использует **все patch-токены**. Каждый патч теперь — потенциальный объект. К каждому прикрепляются две лёгкие «головы»:

- **Classification head:** линейная проекция patch-эмбеддинга в пространство текстовых эмбеддингов + косинусное сходство с текстовыми запросами → оценка класса
- **Box head:** маленький MLP, предсказывающий координаты bounding box [cx, cy, w, h]

[СХЕМА: OWL-ViT — Image → ViT (без CLS pooling) → N×N patch tokens → [Classification Head: cos(patch, text)] + [Box Head: MLP → bbox] → NMS → Detections]

#### Как это используется

```
Текстовые запросы: ["a cat", "a remote control", "a couch"]
                          ↓ Text Encoder
                  Текстовые эмбеддинги
                          ↓
Изображение → ViT → Для каждого патча:
                      class_score = cos(patch_emb, text_emb)  ← для каждого запроса
                      bbox = BoxHead(patch_emb)               ← координаты рамки
                          ↓
                  Фильтрация по порогу + NMS (подавление дубликатов)
                          ↓
                  Результат: [{label: "cat", score: 0.95, box: [x,y,w,h]}, ...]
```

**OWLv2** добавляет **objectness head** — предсказывает, содержит ли данный патч **какой-либо** объект (независимо от запроса). Это позволяет фильтровать фон до классификации. Также масштабируется через **self-training**: существующий детектор генерирует pseudo-annotations на неразмеченных данных → до 1B+ примеров.

> Источники: Minderer et al. — *Simple Open-Vocabulary Object Detection with Vision Transformers* (ECCV 2022); Minderer et al. — *Scaling Open-Vocabulary Object Detection* (NeurIPS 2023)


### 3.7 Проблемы ZSL в CV

#### Сдвиг домена (Domain Shift)

Проекция (матрица $W$), обученная на seen-классах, **систематически сдвигает** спроецированные точки unseen-классов от их истинных семантических прототипов. Модель «привыкает» к распределению видимых классов и плохо обобщается на невидимые.

#### Проблема «хабов» (Hubness Problem)

В высокоразмерных пространствах (2048 измерений у ResNet-признаков) возникает статистический артефакт: некоторые точки становятся **ближайшими соседями непропорционально большого числа** других точек — это «хабы». В контексте ZSL это значит, что некоторые unseen-классы «перетягивают» на себя почти все предсказания.

Одно из решений: вместо проекции визуального пространства в семантическое (как делают DAP, DeViSE), делать **обратную проекцию** — семантическое в визуальное. Обратная проекция подавляет hubness.

#### Смещение к видимым классам в GZSL

В Generalized ZSL модель систематически **предпочитает seen-классы**, потому что обучалась именно на них. Генеративные подходы (f-CLSWGAN) частично решают эту проблему, добавляя синтетические данные для unseen-классов.

| COCO | 80 категорий | — | 330K | ZS Object Detection | — |

Стандартные разбиения (Proposed Splits) предложены в обзоре Xian et al. (TPAMI 2018). Важный момент: unseen-классы **не должны входить** в ImageNet 1K (на котором предобучен ResNet), иначе оценка нечестная — модель «видела» эти классы через предобученный backbone.

Визуальные признаки: обычно 2048-мерные векторы из **ResNet-101**, предобученного на ImageNet 1K. Метрики: per-class averaged top-1 accuracy (ZSL), гармоническое среднее $H$ (GZSL).

## 4. Zero-Shot Learning в NLP

### 4.1 Предыстория: «бездатная» классификация (Dataless Classification)

Ещё до появления термина «zero-shot learning» в NLP существовал подход «dataless classification» (классификация без данных). Идея: классифицировать тексты через **семантическое сходство** между документом и названием класса, без какой-либо обучающей выборки.

Инструмент — **ESA (Explicit Semantic Analysis)**: каждый текстовый фрагмент (документ или название класса) отображается в высокоразмерное пространство концептов (статей Википедии). Классификация — просто поиск ближайшего названия класса в этом пространстве:

$$\hat{y} = \arg\min_i \|\varphi(l_i) - \varphi(d)\|$$

где $\varphi(\cdot)$ — ESA-представление, $l_i$ — название $i$-го класса, $d$ — документ.

Результат: **без единого размеченного примера** достигается 85.3% на 20 Newsgroups — это лучше, чем supervised Naive Bayes с 10 размеченными примерами (71.7%).

> Источник: Chang, Ratinov, Roth, Srikumar — *Importance of Semantic Representation: Dataless Classification* (AAAI 2008)

### 4.2 NLI-подход к zero-shot классификации текстов

#### Ключевая идея

Любая задача классификации текстов может быть сведена к задаче **textual entailment** (текстового следования): «следует ли из текста $x$, что он относится к классу $y$?»

Для этого используется модель, обученная на задаче NLI (Natural Language Inference). NLI — это задача определения отношения между двумя предложениями (premise и hypothesis): **entailment** (следование), **contradiction** (противоречие) или **neutral** (нейтральное).

#### Как это работает

Алгоритм zero-shot классификации через NLI:

1. Входной текст $x$ становится **premise** (посылка)
2. Для каждой метки $y$ формируется **hypothesis** (гипотеза) по шаблону:
   - Для темы: «This text is about {$y$}»
   - Для эмоции: «This text expresses {$y$}»
   - Для ситуации: «The people there need {$y$}»
3. NLI-модель вычисляет вероятность entailment для каждой пары:
   $p(\text{entailment} | x, h_y)$
4. Выбирается класс с максимальной вероятностью:

$$\hat{y} = \arg\max_{y \in \mathcal{Y}} p(\text{entailment} | x, h_y)$$

**Пример:**

Текст: «Apple reported record quarterly revenue of $124 billion»

| Гипотеза | p(entailment) |
|----------|--------------|
| «This text is about Business» | **0.92** |
| «This text is about Sci/Tech» | 0.06 |
| «This text is about Sports» | 0.01 |
| «This text is about World» | 0.01 |

Предсказание: **Business** ✓

> Источник: Yin, Hay, Roth — *Benchmarking Zero-shot Text Classification: Datasets, Evaluation and Entailment Approach* (EMNLP 2019)

### 4.3 Модели для NLI-based ZSL

#### BART-large-MNLI (~400M параметров)

BART — encoder-decoder модель (bidirectional encoder + autoregressive decoder), предложенная Facebook. BART-large-MNLI — это BART-large, дообученный на задаче NLI (датасет MultiNLI, ~433 000 пар предложений с тремя метками: entailment/neutral/contradiction).

Для zero-shot инференса: входной текст — premise, для каждой метки формируется hypothesis «This example is {label}». Модель выдаёт logits для трёх классов [contradiction, neutral, entailment]. Значение neutral отбрасывается, softmax по contradiction и entailment, вероятность entailment — оценка принадлежности метке.

#### DeBERTa-v3-MNLI (~300–400M параметров)

DeBERTa (Decoding-enhanced BERT with disentangled attention) внедряет **разделённое внимание** (disentangled attention): каждый токен представлен **двумя** отдельными векторами — один для содержания (что это за слово), другой для позиции (где оно стоит).

Стандартный attention в Transformer вычисляет одну оценку внимания. В DeBERTa оценка внимания $A_{i,j}$ состоит из **трёх компонент**:

$$A_{i,j} = H_i H_j^\top + H_i P_{j|i}^\top + P_{i|j} H_j^\top$$

Здесь:
- $H_i H_j^\top$ — **содержание↔содержание**: насколько слова семантически связаны
- $H_i P_{j|i}^\top$ — **содержание↔позиция**: насколько содержание $i$-го токена зависит от позиции $j$-го
- $P_{i|j} H_j^\top$ — **позиция↔содержание**: насколько позиция $i$-го токена связана с содержанием $j$-го

Это даёт более тонкое моделирование зависимостей в тексте.

DeBERTa-v3 дополнительно заменяет MLM (Masked Language Modeling) на **RTD** (Replaced Token Detection, в стиле ELECTRA): вместо «предскажи замаскированное слово» — «определи, какие слова были заменены». Это более эффективный способ предобучения.

Модели MoritzLaurer/deberta-v3-large-zeroshot-v2.0, обученные на 5+ NLI-датасетах и 27+ задачах классификации (переформулированных как NLI), значительно превосходят bart-large-mnli на задачах zero-shot.

### 4.4 GPT-3 и In-Context Learning (обучение в контексте)

В 2020 году GPT-3 продемонстрировал принципиально иной подход к zero-shot и few-shot learning: **без обновления весов модели вообще**.

GPT-3 (175 миллиардов параметров, 96 слоёв Transformer, ширина 12288) выполняет задачи в трёх режимах:

- **Zero-shot:** только текстовая инструкция, без примеров — «Определи тональность текста: "Фильм потрясающий!" → »
- **One-shot:** инструкция + один пример — «"Фильм ужасный" → Негативная. "Фильм потрясающий!" → »
- **Few-shot:** инструкция + K примеров (ограничено контекстным окном ~2048 токенов)

Ключевое отличие от классического ZSL: GPT-3 не использует явного семантического пространства (атрибуты, word2vec). Модель опирается на **знания, полученные при предобучении** на 300+ миллиардах токенов. Производительность **масштабируется с размером модели** — чем больше модель, тем лучше работает zero/few-shot.

Два теоретических объяснения, почему это работает:

**Имплицитный байесовский вывод:** при предобучении модель видит тексты из разных «доменов» (концептов). Когда пользователь даёт промпт с примерами, модель **распознаёт скрытый концепт** (например, «это задача определения тональности») и генерирует ответ из соответствующего распределения.

**Имплицитный gradient descent:** Transformer-attention при обработке промпта фактически выполняет нечто **похожее на шаг градиентного спуска**. Примеры в контексте работают как мини-обучающая выборка, а forward pass — как одна итерация обучения.

> Источники: Brown et al. — *Language Models are Few-Shot Learners* (NeurIPS 2020); Xie et al. — *An Explanation of In-context Learning as Implicit Bayesian Inference* (ICLR 2022); Dai et al. — *Why Can GPT Learn In-Context?* (ACL 2023)

## 5. Few-Shot Learning в Computer Vision

### 5.1 Подходы на основе обучения метрики (Metric Learning)

Общая идея всех metric learning подходов: **научить нейросеть такое пространство признаков, в котором похожие объекты расположены близко, а непохожие — далеко.** После этого классификация нового объекта — просто поиск ближайшего соседа (или ближайшего прототипа).

Все методы в этом разделе **не обучают классификатор для конкретных классов**. Вместо этого они учат **меру сходства**, которая обобщается на любые новые классы.

#### 5.1.1 Siamese Networks (Сиамские сети)

**Задача:** определить, принадлежат ли два изображения **одному классу** — не распознать конкретный класс, а ответить «да/нет» на вопрос о сходстве.

**Почему «сиамские»:** две одинаковые CNN с **общими весами** (как сиамские близнецы) обрабатывают два изображения параллельно, и результаты сравниваются.

**Архитектура:**

[СХЕМА: Siamese Networks — Image₁ → CNN (shared weights) → h₁ → |h₁ − h₂| (поэлементная разность) → FC → σ → p ∈ [0,1]; Image₂ → CNN (shared weights) → h₂ ↗]

Конкретная реализация для Omniglot (рукописные символы, изображения 105×105):

- **Энкодер** (одинаковый для обоих входов): 4 свёрточных слоя (64→128→128→256 фильтров), каждый с ReLU и MaxPooling
- **Сравнение:** взвешенная поэлементная L1-разность между эмбеддингами двух изображений
- **Выход:** один нейрон с sigmoid → вероятность $p \in [0, 1]$, что два изображения из одного класса

Формула выхода:

$$p(x_1, x_2) = \sigma\left(\sum_j \alpha_j \cdot |h_1^{(j)} - h_2^{(j)}|\right)$$

Здесь:
- $h_1, h_2 \in \mathbb{R}^{4096}$ — эмбеддинги двух изображений после CNN
- $|h_1^{(j)} - h_2^{(j)}|$ — абсолютная разность по $j$-й компоненте
- $\alpha_j$ — обучаемый вес для $j$-й компоненты
- $\sigma$ — sigmoid-функция

**Функция потерь** — бинарная кросс-энтропия:

$$L(x_1, x_2) = y \cdot \log p(x_1, x_2) + (1 - y) \cdot \log(1 - p(x_1, x_2))$$

где $y = 1$, если оба изображения из одного класса, $y = 0$ — если из разных.

**One-shot классификация:** для нового изображения $x^*$ сравниваем его со всеми примерами в support set и выбираем класс с максимальным сходством:

$$\hat{y} = \arg\max_c \; p(x^*, x_c)$$

где $x_c$ — пример класса $c$ из support set.

> Источник: Koch, Zemel, Salakhutdinov — *Siamese Neural Networks for One-shot Image Recognition* (ICML Workshop, 2015)

#### 5.1.2 Matching Networks (Сопоставляющие сети)

**Отличие от Siamese Networks:** вместо попарного сравнения «query vs один support» — сравнение **query со всем support set одновременно** через механизм внимания (attention).

**Идея:** предсказание для нового изображения — **взвешенное голосование** по support set:

$$\hat{y} = \sum_{i=1}^{|S|} a(\hat{x}, x_i) \cdot y_i$$

Здесь $a(\hat{x}, x_i)$ — вес внимания (насколько query $\hat{x}$ «похож» на support-пример $x_i$), $y_i$ — метка support-примера. Веса вычисляются через softmax по косинусным сходствам:

$$a(\hat{x}, x_i) = \frac{\exp\big(\cos(f(\hat{x}),\; g(x_i))\big)}{\sum_j \exp\big(\cos(f(\hat{x}),\; g(x_j))\big)}$$

Здесь:
- $f(\hat{x})$ — эмбеддинг query-изображения
- $g(x_i)$ — эмбеддинг support-изображения
- $\cos(\cdot, \cdot)$ — косинусное сходство

**Ключевая инновация — Full Context Embeddings (FCE):** support-примеры кодируются **не независимо**, а с учётом всего support set через двунаправленный LSTM:

$$g(x_i, S) = \overrightarrow{h}_i + \overleftarrow{h}_i + g'(x_i)$$

где $g'(x_i)$ — «сырой» эмбеддинг из CNN, а $\overrightarrow{h}_i, \overleftarrow{h}_i$ — скрытые состояния bi-LSTM, обработавшего эмбеддинги всех support-примеров. Это позволяет каждому support-примеру «знать» о других и адаптировать своё представление.

**Эпизодическое обучение** впервые формализовано именно здесь. Авторы сформулировали принцип: **«условия обучения должны совпадать с условиями тестирования»** (test and train conditions must match). Каждый мини-батч при обучении — эпизод, имитирующий N-way K-shot задачу.

[СХЕМА: Matching Networks — Support set {(x₁,y₁),...,(xₖ,yₖ)} → bi-LSTM → g(xᵢ, S); Query x̂ → f(x̂) → cos similarity с каждым g(xᵢ) → softmax → weights a → Σ aᵢyᵢ → ŷ]

> Источник: Vinyals et al. — *Matching Networks for One Shot Learning* (NeurIPS, 2016)


#### 5.1.3 Prototypical Networks (Прототипные сети)

Это центральный метод FSL — одновременно простой, эффективный и теоретически обоснованный. Разбираем подробно.

**Идея в одном предложении:** каждый класс представлен **одной точкой** в пространстве эмбеддингов — **прототипом**, который вычисляется как среднее эмбеддингов support-примеров. Новый пример относится к тому классу, чей прототип ближе.

**Почему это работает (интуиция):** если нейросеть научилась хорошему пространству признаков, то изображения одного класса группируются в кластеры. Центр кластера (среднее) — естественный «представитель» класса. Даже если у нас всего 5 примеров класса, их среднее — разумная оценка центра кластера.

**Архитектура Embedding Network $f_\phi$:**

В оригинальной статье используется намеренно простая CNN из 4 блоков — авторы хотели показать, что идея прототипов работает даже с простым энкодером.

Для miniImageNet (изображения 84×84×3):

```
Block 1: Conv2d(3, 64, kernel=3, padding=1) → BatchNorm → ReLU → MaxPool(2)    → 42×42×64
Block 2: Conv2d(64, 64, kernel=3, padding=1) → BatchNorm → ReLU → MaxPool(2)   → 21×21×64
Block 3: Conv2d(64, 64, kernel=3, padding=1) → BatchNorm → ReLU → MaxPool(2)   → 10×10×64
Block 4: Conv2d(64, 64, kernel=3, padding=1) → BatchNorm → ReLU → MaxPool(2)   → 5×5×64
                                                           Flatten → 1600-мерный вектор
```

Всего ~113K параметров. Никаких полносвязных слоёв. Никакой classification head. Модель **не имеет понятия «класс»** — она просто переводит изображение в вектор.

[СХЕМА: Prototypical Networks — Support set → fφ(каждый пример) → Embeddings → mean по классам → Прототипы c₁, c₂, ..., cₙ; Query x* → fφ(x*) → Расстояния до прототипов → softmax → вероятности классов]

**Математика:**

**Шаг 1. Вычисление прототипа каждого класса** — среднее эмбеддингов support-примеров:

$$\mathbf{c}_k = \frac{1}{|S_k|} \sum_{(x_i, y_i) \in S_k} f_\phi(x_i)$$

Здесь:
- $S_k$ — множество support-примеров класса $k$ (их $K$ штук)
- $f_\phi(x_i) \in \mathbb{R}^M$ — эмбеддинг $i$-го изображения (выход CNN)
- $\mathbf{c}_k \in \mathbb{R}^M$ — прототип класса $k$ (центр кластера)

**Шаг 2. Классификация query-примера** — softmax по отрицательным расстояниям до прототипов:

$$p_\phi(y = k \mid x^*) = \frac{\exp(-d(f_\phi(x^*),\; \mathbf{c}_k))}{\sum_{k'} \exp(-d(f_\phi(x^*),\; \mathbf{c}_{k'}))}$$

Здесь:
- $x^*$ — query-изображение, которое нужно классифицировать
- $d(\cdot, \cdot)$ — функция расстояния (евклидово расстояние)
- Числитель: $\exp$ от отрицательного расстояния до прототипа класса $k$ — чем ближе, тем больше
- Знаменатель: сумма по всем классам — нормализация до вероятностей

**Шаг 3. Функция потерь** — отрицательный логарифм вероятности правильного класса (NLL):

$$J(\phi) = -\log\; p_\phi(y = k \mid x^*)$$

Потери считаются для каждого query-примера в эпизоде, усредняются и используются для обновления весов $\phi$ энкодера через backpropagation.

**Теоретическое обоснование: связь с дивергенциями Брэгмана**

Это центральный теоретический результат статьи. Авторы показали, что Prototypical Networks — не просто эвристика, а **эквивалент оценки плотности смеси** (mixture density estimation).

**Дивергенция Брэгмана** — обобщение понятия «расстояния», определяемое через строго выпуклую функцию $\varphi$:

$$d_\varphi(z, z') = \varphi(z) - \varphi(z') - (z - z')^\top \nabla\varphi(z')$$

Частные случаи:
- При $\varphi(z) = \|z\|^2$ получаем **квадрат евклидова расстояния**
- При $\varphi(z) = \sum z_i \log z_i$ получаем **KL-дивергенцию**

**Теорема (Banerjee et al., 2005):** для дивергенций Брэгмана оптимальный представитель кластера (точка, минимизирующая суммарное расстояние до всех точек кластера) — это **среднее**.

Следствие: если мы используем евклидово расстояние (дивергенцию Брэгмана для $\varphi(z) = \|z\|^2$), то прототип $\mathbf{c}_k = \text{mean}(S_k)$ — **теоретически оптимальный** представитель класса.

Далее, любое распределение из **экспоненциального семейства** может быть записано через дивергенцию Брэгмана:

$$p_\psi(z|\theta) = \exp\{-d_\varphi(z, \mu(\theta)) - g_\varphi(z)\}$$

Для равновесной смеси таких распределений (все классы равновероятны) вероятность принадлежности к классу $k$:

$$p(y = k \mid z) = \frac{\exp(-d_\varphi(z, \mu_k))}{\sum_{k'} \exp(-d_\varphi(z, \mu_{k'}))}$$

Это **тождественно** формуле Prototypical Networks. То есть Prototypical Networks — это эквивалент байесовского классификатора для смеси распределений экспоненциального семейства.

**Почему евклидово расстояние, а не косинусное сходство:**

1. Квадрат евклидова расстояния — это дивергенция Брэгмана → среднее как прототип **теоретически обосновано**
2. Косинусное сходство **не является** дивергенцией Брэгмана → использование среднего как прототипа **не имеет теоретического обоснования**
3. Эмпирически евклидово расстояние **значительно превосходит** косинусное (авторы показывают это в экспериментах)

**Бонус: эквивалентность линейной модели.** При евклидовом расстоянии ProtoNet оказывается эквивалентен **линейному классификатору**:

$$-\|f_\phi(x) - \mathbf{c}_k\|^2 = 2\mathbf{c}_k^\top f_\phi(x) - \mathbf{c}_k^\top \mathbf{c}_k + \text{const}$$

Это линейная функция от $f_\phi(x)$ с весами $2\mathbf{c}_k$ и смещением $-\mathbf{c}_k^\top \mathbf{c}_k$. Нелинейность «спрятана» внутри энкодера $f_\phi$.

**Результаты:** miniImageNet 5-way 1-shot: 49.42 ± 0.78%; 5-way 5-shot: 68.20 ± 0.66%.

**Важная деталь обучения:** авторы обнаружили, что обучение с **более высоким N** (например, 20-way или 30-way), чем при тестировании (5-way), «чрезвычайно полезно» — это вынуждает модель принимать более тонкие решения в пространстве эмбеддингов.

> Источник: Snell, Swersky, Zemel — *Prototypical Networks for Few-shot Learning* (NeurIPS 2017)


#### 5.1.4 Relation Networks (Сети отношений)

**Отличие от предыдущих:** Siamese Networks используют фиксированную L1-разность, Matching Networks — фиксированное косинусное сходство, Prototypical Networks — фиксированное евклидово расстояние. Relation Networks заменяют фиксированную метрику **обучаемой нейросетью**.

**Архитектура** состоит из двух модулей:

1. **Embedding module** $f_\varphi$ — CNN, кодирующая изображения в feature maps (не векторы, а карты признаков!)
2. **Relation module** $g_\phi$ — нейросеть, принимающая **конкатенацию** двух feature maps и выдающая скаляр $r \in [0, 1]$ — оценку сходства

$$r_{i,j} = g_\phi\Big(\text{concat}\big(f_\varphi(x_i),\; f_\varphi(x_j)\big)\Big)$$

Для K > 1 (несколько примеров на класс): feature maps support-примеров одного класса **суммируются поэлементно** перед конкатенацией с query.

**Функция потерь** — MSE (не кросс-энтропия!):

$$L = \sum_{i,j} (r_{i,j} - \mathbb{1}(y_i = y_j))^2$$

Здесь $\mathbb{1}(y_i = y_j) = 1$, если $x_i$ и $x_j$ из одного класса, и $0$ иначе. MSE заставляет relation module выдавать значения близкие к 1 для одинаковых классов и к 0 для разных.

**Результат:** miniImageNet 5-way 1-shot: 50.44 ± 0.82%.

> Источник: Sung et al. — *Learning to Compare: Relation Network for Few-Shot Learning* (CVPR 2018)


### 5.2 Подходы на основе мета-обучения (Meta-Learning)

Metric learning учит **хорошее пространство признаков**. Meta-learning подходит с другой стороны: учит **хорошую начальную точку** для быстрой адаптации к новой задаче.

#### 5.2.1 MAML (Model-Agnostic Meta-Learning)

**Идея:** найти такую **инициализацию весов** $\theta$, из которой модель может адаптироваться к любой новой задаче за **несколько шагов градиентного спуска** на малом количестве данных.

Обычное обучение: случайная инициализация → тысячи шагов градиентного спуска → модель для одной задачи.
MAML: **мета-обучение инициализации** → 1–5 шагов градиентного спуска на новой задаче → рабочая модель.

**Два уровня оптимизации:**

**Inner loop (внутренний цикл) — адаптация к конкретной задаче:**

Для задачи $T_i$ (один эпизод: N-way K-shot) делаем один или несколько шагов SGD на support set:

$$\theta'_i = \theta - \alpha \nabla_\theta L_{T_i}(f_\theta)$$

Здесь:
- $\theta$ — текущие «мета-веса» (общая инициализация)
- $\alpha$ — learning rate внутреннего цикла
- $L_{T_i}(f_\theta)$ — loss на support set задачи $T_i$
- $\theta'_i$ — адаптированные веса для задачи $T_i$

**Outer loop (внешний цикл) — мета-оптимизация:**

Обновляем мета-веса $\theta$ так, чтобы **после адаптации** (inner loop) модель хорошо работала на query set:

$$\theta \leftarrow \theta - \beta \nabla_\theta \sum_{T_i \sim p(T)} L_{T_i}(f_{\theta'_i})$$

Здесь:
- $\beta$ — learning rate внешнего цикла (мета-learning rate)
- $L_{T_i}(f_{\theta'_i})$ — loss на **query set** задачи $T_i$, вычисленный с **адаптированными** весами $\theta'_i$
- Сумма по нескольким задачам из батча

**Ключевая сложность:** внешний цикл оптимизирует $\theta$, но loss вычисляется на $\theta'_i = \theta - \alpha \nabla_\theta L$. Это означает **вычисление градиента через градиент** (second-order derivatives, Hessian-vector products). Вычислительно дорого.

[СХЕМА: MAML — θ (мета-веса) → [Inner loop: θ − α∇L → θ'₁ для задачи T₁; θ − α∇L → θ'₂ для задачи T₂; ...] → [Outer loop: оценка θ'ᵢ на query sets → ∇θ ΣL(fθ'ᵢ) → обновление θ]]

**FOMAML (First-Order MAML):** упрощение, которое **отбрасывает вторые производные**. Удивительный результат: производительность почти не отличается от полного MAML, при этом на ~33% быстрее. Объяснение: для ReLU-сетей вторые производные почти везде равны нулю (ReLU — кусочно-линейная функция).

**Результат:** miniImageNet 5-way 1-shot: 48.70 ± 1.84%; 5-shot: 63.11 ± 0.92%.

> Источник: Finn, Abbeel, Levine — *Model-Agnostic Meta-Learning for Fast Adaptation of Deep Networks* (ICML 2017)


#### 5.2.2 Reptile

**Проблема MAML:** даже FOMAML требует дифференцирования через несколько шагов оптимизации. Reptile — ещё более простая альтернатива.

**Алгоритм:**

```
Инициализируем φ
Для каждой итерации:
    Выбираем задачу τ
    φ̃ = SGD(φ, задача τ, k шагов)    ← делаем k шагов обучения на задаче
    φ ← φ + ε(φ̃ − φ)                 ← сдвигаемся в сторону результата
```

Всё. Не нужно дифференцирование через оптимизацию. Просто: обучись на задаче несколько шагов → сдвинь начальную точку в сторону результата.

При $k = 1$ Reptile вырождается в обычный SGD на ожидаемом loss (ничего мета-). При $k \geq 2$ появляются **члены высших порядков**, которые и дают мета-обучение.

**Связь с MAML** через разложение Тейлора (при $k = 2$):

$$\mathbb{E}[g_{\text{MAML}}] = (1) \cdot \text{AvgGrad} - (2\alpha) \cdot \text{AvgGradInner}$$
$$\mathbb{E}[g_{\text{Reptile}}] = (2) \cdot \text{AvgGrad} - (\alpha) \cdot \text{AvgGradInner}$$

Здесь $\text{AvgGradInner} = \mathbb{E}[H_2 g_1]$ — член, максимизирующий **внутреннее произведение градиентов** разных мини-батчей одной задачи. Этот член поощряет обобщение **внутри задачи** — именно то, что нужно для few-shot.

Все три алгоритма (MAML, FOMAML, Reptile) содержат одни и те же два члена, различаясь только коэффициентами.

> Источник: Nichol, Achiam, Schulman — *On First-Order Meta-Learning Algorithms* (arXiv:1803.02999, 2018)


### 5.3 CLIP как Few-Shot Learner

CLIP — не только zero-shot модель. Имея предобученный CLIP, можно адаптировать его к новым задачам с несколькими примерами тремя способами:

**Linear Probe (линейный зонд):** заморозить CLIP-энкодер, извлечь эмбеддинги для K примеров каждого класса, обучить **линейный классификатор** (логистическую регрессию) поверх них. Просто и эффективно, но не использует текстовые знания CLIP.

**CoOp (Context Optimization):** вместо ручных промптов «A photo of a {class}» обучить **мягкие непрерывные векторы** контекста. Промпт выглядит как $[V_1][V_2]\ldots[V_n][\text{CLASS}]$, где $V_i$ — обучаемые 512-мерные векторы. Обучается только 16 векторов × 512 dim ≈ 8K параметров, всё остальное заморожено. Улучшает few-shot accuracy на 10–15% поверх ручных промптов.

**Tip-Adapter (Training-free):** кэшировать few-shot примеры в виде пар (CLIP-эмбеддинг, метка), использовать их как ключи для **attention-weighted** классификации нового примера. Не требует обучения вообще — работает сразу.

## 6. Few-Shot Learning в NLP

### 6.1 SetFit (Sentence Transformer Fine-tuning)

**Проблема:** существующие few-shot методы для NLP (PET, T-Few) либо требуют ручных промптов, либо используют модели с миллиардами параметров (GPT-3: 175B, T-Few: 11B). Это непрактично.

**Решение SetFit:** двухэтапный подход, не требующий ни промптов, ни гигантских моделей. Работает с моделями на порядок меньше (~110M параметров) и при этом конкурирует с GPT-3.

#### Этап 1: Контрастивное дообучение Sentence Transformer

Из $K$ размеченных примеров на класс генерируются **пары предложений**:
- **Позитивные пары** $(x_i, x_j)$: оба из одного класса → метка сходства = 1
- **Негативные пары** $(x_i, x_k)$: из разных классов → метка сходства = 0

По умолчанию генерируется $R = 20$ пар на класс. Из 8 примеров на класс это даёт ~160 пар — в 20 раз больше, чем исходных примеров. Это ключевой механизм: **умножение данных** через комбинаторику пар.

Sentence Transformer дообучается с **Cosine Similarity Loss**:

$$\mathcal{L} = \sum_{(x_i, x_j, y)} \left(\cos(f_\theta(x_i),\; f_\theta(x_j)) - y\right)^2$$

Здесь:
- $f_\theta(x_i) \in \mathbb{R}^{768}$ — эмбеддинг предложения (выход Sentence Transformer)
- $\cos(\cdot, \cdot)$ — косинусное сходство
- $y \in \{0, 1\}$ — метка пары (одинаковый класс = 1, разные = 0)

Loss заставляет ST сближать эмбеддинги текстов одного класса и разводить эмбеддинги разных классов.

#### Этап 2: Обучение классификационной головы

На эмбеддингах дообученного ST обучается **логистическая регрессия**:

$$p(y = k \mid x) = \text{softmax}(\mathbf{W} \cdot f_\theta(x) + \mathbf{b})_k$$

Здесь:
- $f_\theta(x)$ — эмбеддинг текста из дообученного ST
- $\mathbf{W}, \mathbf{b}$ — веса и смещение логистической регрессии (обучаются за секунды)

[СХЕМА: SetFit — K примеров → Генерация пар (позитивные/негативные) → Contrastive fine-tuning Sentence Transformer → Дообученный ST → Embeddings → Logistic Regression head → Classifier]

**Результаты:** SetFit с ~355M параметрами (all-roberta-large-v1) **превосходит GPT-3** (175B) на RAFT benchmark: 71.3% vs 62.7%. При этом в **500 раз** меньше параметров.

> Источник: Tunstall, Reimers et al. — *Efficient Few-Shot Learning Without Prompts* (arXiv:2209.11055, 2022)


### 6.2 Pattern-Exploiting Training (PET)

**Идея:** превратить любую задачу классификации в **cloze-задачу** (задачу заполнения пропуска), которую языковая модель уже умеет решать.

Для этого определяется **Pattern-Verbalizer Pair (PVP)**:

- **Pattern (шаблон):** преобразует вход в предложение с [MASK]. Для задачи определения тональности: $P(x) =$ «$x$. It was [MASK].»
- **Verbalizer (вербализатор):** отображает метки классов в слова из словаря модели. $v(\text{positive}) =$ «great», $v(\text{negative}) =$ «terrible»

**Инференс:** модель заполняет [MASK] с помощью MLM (Masked Language Modeling). Вероятность класса — вероятность соответствующего слова:

$$q_P(y \mid x) = \text{softmax по меткам}\big(\text{MLM logit для } v(y) \text{ на позиции [MASK] в } P(x)\big)$$

**Трёхшаговый процесс PET:**

1. Обучается отдельная модель для каждого PVP (разные шаблоны, разные вербализаторы) на малой размеченной выборке
2. Ансамбль моделей создаёт **мягкие метки** (soft labels) для неразмеченных данных (с температурой $T = 2$ для сглаживания)
3. Финальный классификатор обучается на soft-labeled данных

**iPET** итерирует этот процесс: модели поколения $g-1$ аннотируют данные для обучения моделей поколения $g$, постепенно увеличивая объём данных.

**Результат:** PET с ALBERT (235M параметров) **превосходит GPT-3 (175B)** на SuperGLUE с 32 примерами на задачу.

> Источник: Schick, Schütze — *Exploiting Cloze Questions for Few Shot Text Classification and Natural Language Inference* (EACL 2021)


### 6.3 In-Context Learning: теория и связь с FSL

В разделе 4.4 (Часть I) мы рассмотрели, как GPT-3 выполняет задачи через промпт. Здесь углубимся в теоретическое понимание.

**ICL vs классический FSL:**

| Аспект | Классический FSL (ProtoNet, MAML) | In-Context Learning (GPT-3) |
|--------|----------------------------------|----------------------------|
| Обновление весов | Да (при мета-обучении) | Нет (только forward pass) |
| Механизм | Metric learning / мета-оптимизация | Attention к примерам в контексте |
| Размер модели | ~1M–100M | 175B+ |
| Новые задачи | Нужно мета-обучение | Работает из коробки |
| Что адаптируется | Веса энкодера | Ничего (только контекст) |

**ICL как имплицитный gradient descent:** для упрощённого случая (линейный attention) можно показать, что ICL-обновление эквивалентно:

$$F_{\text{ICL}}(q) = (W_{\text{ZSL}} + \Delta W_{\text{ICL}}) \cdot q$$

где $\Delta W_{\text{ICL}} = \sum_i (W_V \cdot x'_i) \otimes (W_K \cdot x'_i)$ — «обновление весов» от демонстрационных примеров. Это двойственно к fine-tuning, где $\Delta W_{\text{FT}}$ получается через backpropagation.

Ключевое отличие: ICL выполняет адаптацию **за один forward pass**, без обратного распространения ошибки.


### 6.4 PEFT-подходы для few-shot адаптации (Parameter-Efficient Fine-Tuning)

Когда есть предобученная большая модель и несколько примеров новой задачи, **полное дообучение** (fine-tuning всех параметров) часто ведёт к переобучению и требует много памяти. PEFT-методы обучают **малую долю параметров**, оставляя основную модель замороженной.

#### 6.4.1 LoRA (Low-Rank Adaptation)

**Идея:** вместо обновления полной матрицы весов $W_0 \in \mathbb{R}^{d \times k}$ обучается **низкоранговое разложение** обновления:

$$W = W_0 + \Delta W = W_0 + B \cdot A$$

Здесь:
- $W_0 \in \mathbb{R}^{d \times k}$ — исходные предобученные веса (**замороженные**, не обучаются)
- $B \in \mathbb{R}^{d \times r}$ — обучаемая матрица «расширения»
- $A \in \mathbb{R}^{r \times k}$ — обучаемая матрица «сжатия»
- $r \ll \min(d, k)$ — ранг (обычно $r = 4$ или $r = 8$)

**Forward pass:**

$$h = W_0 x + \frac{\alpha}{r} B A x$$

где $\alpha$ — масштабирующий коэффициент.

[СХЕМА: LoRA — Вход x → [W₀ (заморожена, d×k)] → h₀; параллельно x → [A (r×k)] → [B (d×r)] → ΔW·x; h = h₀ + (α/r)·ΔW·x → Выход h]

**Инициализация:** $A$ — случайное гауссово, $B$ — нули. Это означает, что в начале обучения $\Delta W = BA = 0$ — модель стартует с исходных предобученных весов.

**При инференсе:** $W = W_0 + BA$ вычисляется **явно** один раз → **нулевая дополнительная задержка** по сравнению с исходной моделью.

**Масштаб экономии:** для GPT-3 (175B параметров) LoRA сокращает число обучаемых параметров в **10 000 раз**, потребление GPU-памяти — в **3 раза**.

> Источник: Hu et al. — *LoRA: Low-Rank Adaptation of Large Language Models* (ICLR 2022)


#### 6.4.2 Adapters (Адаптеры)

**Идея:** вставить маленькие **bottleneck-модули** внутрь каждого слоя Transformer. Основная модель замораживается, обучаются только адаптеры.

Структура одного адаптера:

$$\text{Adapter}(h) = h + W_{\text{up}} \cdot \sigma(W_{\text{down}} \cdot h)$$

Здесь:
- $h$ — входной вектор (скрытое состояние Transformer)
- $W_{\text{down}} \in \mathbb{R}^{m \times d}$ — «сжатие» в bottleneck размерности $m$
- $\sigma$ — нелинейность (ReLU)
- $W_{\text{up}} \in \mathbb{R}^{d \times m}$ — «расширение» обратно
- Сложение с $h$ — residual connection

Два адаптера вставляются в каждый Transformer-слой: один после self-attention, другой после FFN.

При $m = 64$ и BERT-base: добавляется **~3.6% параметров** к модели, результат на GLUE отстаёт от полного fine-tuning всего на **0.4%**.

> Источник: Houlsby et al. — *Parameter-Efficient Transfer Learning for NLP* (ICML 2019)


#### 6.4.3 Prefix Tuning (Настройка префиксов)

**Идея:** добавить обучаемые **непрерывные векторы** (prefix) к keys и values на каждом слое Transformer. Модель не модифицируется — просто на вход attention подаётся расширенная последовательность.

Обучается **~0.1% параметров** модели. Превосходит полное fine-tuning в **low-data режиме** и лучше экстраполирует на невиданные темы.

> Источник: Li, Liang — *Prefix-Tuning: Optimizing Continuous Prompts for Generation* (ACL 2021)

#### Сравнение PEFT-методов

| Метод | Обучаемые параметры | Задержка инференса | Когда выбирать |
|-------|--------------------|--------------------|---------------|
| **LoRA** ($r = 4$) | ~0.1–0.5% | Нулевая (merge) | Основной выбор для few-shot |
| **Adapters** ($m = 64$) | ~3.6% | Добавляется (два модуля на слой) | Когда нужна модульность (разные адаптеры для разных задач) |
| **Prefix Tuning** ($l = 10$) | ~0.1% | Добавляется (удлинение контекста) | Минимум параметров, генеративные задачи |

| **NLP, 8–16 примеров** | SetFit (лучший баланс качество/ресурсы) |
| **NLP, есть предобученная LLM** | LoRA few-shot fine-tuning |
| **NLP, нужно максимальное качество** | PET + ансамбль, или LLM с ICL |
| **Классы появляются динамически** | Prototypical Networks (не надо переобучать) |


### 7.3 Открытые проблемы

**Специализированные домены.** CLIP и SigLIP показывают впечатляющие результаты на «обычных» изображениях, но **деградируют** в специализированных доменах: медицинская визуализация, спутниковые снимки, микроскопия. Причина — эти изображения редки в интернет-данных, на которых обучался CLIP.

**Fine-grained различение.** Отличить лабрадора от золотистого ретривера или один вид птицы от похожего — по-прежнему сложно для zero-shot моделей, даже для CLIP.

**Теоретические гарантии.** Для prompt-based методов (CLIP, ICL) практически отсутствуют формальные гарантии. Почему одна формулировка промпта работает лучше другой — во многом остаётся загадкой.

**Мультимодальное масштабирование.** ImageBind (Meta, 2023) показал, что можно выровнять 6 модальностей (изображения, текст, аудио, глубина, тепло, IMU) в одном пространстве, получая zero-shot способности между любыми парами модальностей. Но масштабирование этого подхода — открытый вопрос.


### 7.4 Связь ZSL/FSL с другими областями ML

ZSL и FSL — не изолированные темы, а **узел**, связывающий множество направлений.

**Transfer Learning (перенос обучения).** ZSL — частный случай transfer learning, где задача-источник (seen-классы) и задача-цель (unseen-классы) разделяют семантическое пространство. CLIP стёр границу между ними: предобучение на (image, text) парах — transfer learning, инференс по промптам — ZSL.

**Metric Learning.** Prototypical Networks, Matching Networks, Siamese Networks — это metric learning: обучить пространство, где сходство объектов отражает их семантическую близость. Та же идея лежит в распознавании лиц (FaceNet, triplet loss), поиске изображений (image retrieval), верификации подписей.

**Генеративные модели.** Связь двусторонняя. f-CLSWGAN *использует* GAN для помощи ZSL. А современные генеративные модели (Stable Diffusion) *демонстрируют* zero-shot способности: генерируют «астронавт верхом на лошади», хотя такого изображения почти наверняка не было в обучающих данных. Общий корень — совместное семантическое пространство, где можно комбинировать концепты.

**Композиционное обобщение (Compositional Generalization).** Человек понимает «красный слон» не потому, что видел красных слонов, а потому что может **скомпоновать** «красный» + «слон». ZSL через атрибуты работает по тому же принципу: новый класс = комбинация уже известных свойств.

**In-Context Learning в LLM.** Когда студент отправляет преподавательскую задачу нейросети, и она решает задачу, на которой не обучалась — это связано с ZSL, но это более широкое явление. LLM не использует явного семантического пространства (атрибуты, word2vec). Вместо этого модель при предобучении усвоила **паттерны решения** из учебников, разборов, объяснений — и применяет подходящий паттерн к новой задаче. Это «zero-shot по задачам» (zero-shot task generalization), а не «zero-shot по классам».

**Open-World Detection.** OWL-ViT → Grounding DINO → модели, которые не только распознают заданные классы, но и обнаруживают **«неизвестные» объекты** — то, чего модель вообще не ожидала увидеть. Это ZSL в его самой чистой форме.

**Continual Learning (непрерывное обучение).** GZSL — пересечение ZSL и continual learning: модель должна распознавать и старые, и новые классы, не забывая старые. Bias к seen-классам в GZSL — проявление «катастрофического забывания» (catastrophic forgetting), центральной проблемы continual learning.
